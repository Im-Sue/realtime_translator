# 桌面宿主与交互模块

## 模块定位

桌面宿主与交互模块负责整个桌面应用的壳层能力，包括窗口管理、sidecar 生命周期、前端页面交互和 WebSocket 客户端连接。

## 当前实现

当前实现分布在：

- `src-tauri/src/lib.rs`
- `src-tauri/tauri.conf.json`
- `src/App.tsx`
- `src/hooks/useWebSocket.ts`
- `src/pages/ConfigPage.tsx`
- `src/pages/LogPage.tsx`
- `src/components/StatusBar.tsx`
- `src/components/SubtitleOverlay.tsx`

## 职责

- 启动 Python sidecar。
- 在开发和生产模式之间切换 sidecar 启动方式。
- 创建主窗口和字幕浮窗。
- 管理前端对 `/ws/control`、`/ws/logs`、`/ws/subtitle` 的连接。
- 承接配置、启动、停止、日志查看和字幕展示交互。

## 输入与输出

输入：

- 用户操作
- Tauri 事件
- sidecar 端口
- WebSocket 推送消息

输出：

- 控制命令
- 日志列表展示
- 字幕浮窗展示
- 运行状态展示

## 核心链路

1. 前端调用 `launch_sidecar`。
2. Tauri 拉起 sidecar 并读取端口。
3. `useWebSocket` 建立三路连接。
4. 页面组件消费运行状态、配置、日志和字幕消息。

## 依赖

- Tauri
- React
- 浏览器 WebSocket
- sidecar 进程

## 上下游边界

上游：

- 用户交互

下游：

- sidecar
- Tauri 窗口系统
- WebSocket 服务端

## 当前解耦情况

宿主层与业务核心之间已经通过 sidecar 边界很好地隔离。前端也没有直接触达 Python 细节，而是统一通过 WebSocket 协议交互，这一点是当前架构的重要优点。

## 当前限制

- 前端状态仍偏轻量，主要依赖后端运行态。
- 主窗口当前仍是“配置页/日志页”Tab 切换，尚未形成更细的领域 UI 结构。
