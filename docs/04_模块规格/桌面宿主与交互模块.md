# 桌面宿主与交互模块

## 模块定位

桌面宿主与交互模块负责整个桌面应用的壳层能力，包括 sidecar 生命周期、窗口管理、WebSocket 连接和前端交互，是“桌面 UI”与“Python 引擎”之间的边界层。

## 当前实现

当前实现分布在以下文件：

- `src-tauri/src/lib.rs`
- `src-tauri/tauri.conf.json`
- `src/App.tsx`
- `src/hooks/useWebSocket.ts`
- `src/pages/ConfigPage.tsx`
- `src/pages/LogPage.tsx`
- `src/components/StatusBar.tsx`
- `src/components/SubtitleOverlay.tsx`

## 主要职责

- 启动和管理 Python sidecar
- 管理主窗口与字幕浮窗
- 建立 `/ws/control`、`/ws/logs`、`/ws/subtitle` 三条 WebSocket
- 提供配置编辑、运行控制、日志查看和字幕展示交互
- 在开发模式与打包模式之间切换 sidecar 启动方式

## 核心链路

```text
App.tsx
  -> invoke('launch_sidecar')
  -> Rust 读取 sidecar stdout 端口
  -> useWebSocket 建立 3 条 WS
  -> ConfigPage / LogPage / SubtitleOverlay 消费数据
```

## 当前分支的关键实现点

### Sidecar 拉起策略

`src-tauri/src/lib.rs` 当前会按下面优先级寻找 Python 引擎：

1. 打包资源目录中的 `python/rt-engine.exe`
2. 可执行文件同级目录中的 bundle 资源
3. 开发模式下系统 `python`

Windows 下还额外加了 `CREATE_NO_WINDOW`，避免 sidecar 启动时弹出控制台窗口。

### 三路 WebSocket 分工

`useWebSocket.ts` 当前固定建立：

- `/ws/control`：命令请求与状态事件
- `/ws/logs`：日志流
- `/ws/subtitle`：字幕流

这让前端页面只处理各自职责，不直接耦合 Python 内部对象。

### 主窗口布局

`App.tsx` 当前主窗口采用左右分栏：

- 左侧：`ConfigPage`
- 右侧：`LogPage`

顶部 `StatusBar` 负责：

- 启动 / 停止翻译器
- 展示运行状态和计时
- 显示 / 隐藏字幕浮窗

### 字幕浮窗

字幕浮窗是独立 WebView 窗口，不与主窗口强绑定，支持：

- 独立显示 / 隐藏
- 展开 / 收起高度
- 常驻顶层
- 接收配置广播动态更新字体和透明度

## 上下游边界

### 上游

- 用户点击、表单输入、窗口操作

### 下游

- Tauri Rust 宿主
- Python sidecar
- 本地 WebSocket 服务

## 当前限制

- 当前前端状态管理仍然偏轻量，复杂交互主要依赖后端状态回推。
- 配置页字段还比较“工程向”，更像调试面板而不是产品化设置中心。
- 运行中的错误恢复仍主要依赖后端，前端没有更细粒度的恢复指引。
