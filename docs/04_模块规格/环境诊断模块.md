# 环境诊断模块

## 模块定位

环境诊断模块负责启动前检查和外部依赖连通性验证，避免用户在运行阶段才暴露环境问题。

## 当前实现

当前实现主要位于 `desktop_backend/services.py` 中的 `HealthService`。

## 职责

- 检查 Python 版本是否满足要求。
- 检查关键 Python 依赖是否可导入。
- 检查 FFmpeg 是否可用。
- 检查平台信息。
- 使用真实认证头测试火山引擎 WebSocket 连通性。

## 输入与输出

输入：

- `/ws/control` 的 `env_check`
- `/ws/control` 的 `test_connection`
- 配置管理模块提供的原始火山凭据

输出：

- 环境检查结果字典
- 火山引擎连接测试结果 `{ok, error}`

## 依赖

- `platform`
- `shutil.which`
- `websockets`
- 火山引擎配置

## 上下游边界

上游：

- 前端配置页
- 运行编排前的人工确认

下游：

- 外部 Python 依赖
- FFmpeg 可执行文件
- 火山引擎 WebSocket 服务

## 当前解耦情况

该模块已经与配置管理和运行编排分离，职责比较集中。它最大的价值是把“启动前校验”和“运行时翻译”分开，避免错误定位混乱。

## 当前限制

- 当前只返回简单检查结果，没有形成标准化诊断等级。
- 对网络异常、认证异常、配额异常还没有结构化分类。
