# 字幕处理与展示模块

## 模块定位

字幕处理与展示模块负责将 Channel 2 的实时文本结果转为用户可读的字幕输出，并根据运行模式选择桌面浮窗或 Tkinter 兼容窗口进行展示。

## 当前实现

当前由两部分组成：

1. 字幕处理子层  
位于 `main.py` 中，由 `_append_ch2_fragment()` 和 `_flush_ch2_buffer()` 负责文本聚合、去重和输出时机控制。

2. 字幕展示子层  
位于：

- `src/components/SubtitleOverlay.tsx`
- `gui/subtitle_window.py`

前者用于桌面模式，后者用于 CLI 兼容模式。

## 职责

- 对实时文本片段做英文/中文分流。
- 对片段做去重与句子级聚合。
- 在合适时机 `flush` 成最终字幕。
- 将字幕推送给桌面浮窗或 Tkinter 窗口。

## 输入与输出

输入：

- 火山引擎返回的文本片段

输出：

- `{type, en, zh, is_final}` 字幕消息
- 或 Tkinter 窗口可显示文本

## 核心规则

- 英文和中文分别聚合。
- 英文片段使用更严格的近重复检测。
- 正常情况下按句子边界输出。
- 长时间无新片段时允许强制 `flush`。

## 上下游边界

上游：

- 火山引擎对接模块
- 翻译主控对象

下游：

- `/ws/subtitle`
- React 字幕浮窗
- Tkinter 字幕窗口

## 当前解耦情况

展示层已经支持双实现，边界相对清晰。真正尚未完全解耦的是“字幕处理逻辑”，它目前仍嵌在 `DualChannelTranslator` 内部。

## 当前耦合点

- 字幕聚合规则与翻译主循环放在同一类中。
- 字幕 flush 策略与运行状态密切耦合。

## 后续建议

- 将字幕聚合抽成独立服务对象。
- 保持“字幕处理”和“字幕展示”分别演进。
