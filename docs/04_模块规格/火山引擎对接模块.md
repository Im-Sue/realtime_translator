# 火山引擎对接模块

## 模块定位

火山引擎对接模块负责与火山实时同传服务建立 WebSocket 会话、发送音频并接收翻译结果，是系统访问外部翻译能力的协议适配层。

## 当前实现

当前实现位于 `core/volcengine_client.py`，核心对象包括：

- `VolcengineConfig`
- `TranslationResult`
- `VolcengineTranslator`

## 主要职责

- 建立带认证头的 WebSocket 连接
- 启动和结束翻译会话
- 发送音频块
- 接收文本、音频和会话状态事件
- 解析 protobuf 响应
- 对异常做基础分类与自动恢复
- 输出 `CH1` 音频链路的调试快照

## 输入与输出

### 输入

- 火山配置：`ws_url / app_key / access_key / resource_id`
- 会话参数：`mode / source_language / target_language`
- 音频输出参数：`target_audio_format / target_audio_rate`
- 实时音频块 `bytes`

### 输出

- `TranslationResult`
  - `event`
  - `sequence`
  - `text`
  - `audio_data`
  - `is_finished`
  - `is_failed`
  - `error_message`

## 当前分支的重点变化

### CH1 目标音频格式可配置

当前 `VolcengineTranslator` 已支持：

- `target_audio_format`
- `target_audio_rate`

这让 `CH1` 可以直接向火山请求 `pcm / 48000Hz` 音频，而不再固定走 `ogg_opus / 24000Hz`。

### 首包校验

为了快速发现服务端返回格式与本地播放链路不一致的问题，当前实现会对首个音频包做基础校验：

- `pcm`：字节数必须满足 16bit PCM 的偶数字节特征
- `ogg_opus`：数据包需要以 `OggS` 开头

### 调试快照

模块新增 `get_debug_snapshot()`，会输出：

- 已接收音频包数量
- 音频累计字节数
- 最近音频包大小与序号
- 最近两包之间的时间间隔
- 估算累计音频时长

这些信息会被 `main.py` 的 `channel1_diagnostics_loop()` 汇总打印，用来区分“上游回包稀疏”还是“下游播放断流”。

## 异常与恢复策略

当前实现把错误分成两类：

- 可重试错误：如模型推理错误、超时、网络抖动
- 不可重试错误：如认证失败、配额问题、无效参数

当会话失败且 `auto_reconnect=True` 时，会按指数退避尝试重连并重新启动会话。

## 上下游边界

### 上游

- `DualChannelTranslator`

### 下游

- 火山引擎 WebSocket 服务
- protobuf 消息定义

## 当前限制

- 自动恢复仍然是“会话级重试”，还没有做到音频片段补发或断点续传。
- 错误分类仍以关键字匹配为主，精度依赖外部错误文本格式。
- 供应商接口尚未抽象，如果未来接入其他语音服务，主控层仍需改动。
