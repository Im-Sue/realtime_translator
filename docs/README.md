# docs 知识库说明

`docs/` 采用三层结构管理项目知识，目标是让人和工具都能快速定位信息。

## 目录结构

### Layer 0：协作机制

- `docs/.ccb/index/`：项目轻量索引，启动时快速了解架构、模块和关键决策。
- `docs/.ccb/specs/active/`：进行中的任务规格。
- `docs/.ccb/specs/archive/`：已完成规格归档。
- `docs/.ccb/decisions/`：ADR 架构决策记录。

### Layer 1：开发知识库

- `docs/01_架构设计/`：项目级架构、技术底座、通用规范。
- `docs/02_需求设计/`：需求背景、目标、待确认事项。
- `docs/03_开发计划/`：技术方案、任务拆解、实施路径。
- `docs/04_模块规格/`：模块边界、接口、行为和现状说明。
- `docs/05_经验沉淀/`：踩坑记录、问题复盘、最佳实践。
- `docs/10_接口文档/`：对外接口和协议说明。
- `docs/11_数据库设计/`：表结构、索引和迁移设计。
- `docs/12_数据流设计/`：数据采集、传输、处理链路。
- `docs/13_项目报告/`：阶段报告、验收记录、实施总结。

### Layer 2：归档层

- `docs/99_归档/`：历史版本、废弃方案、低频查阅文档。

## 使用约定

1. 新文档优先放到对应数字目录，不要直接平铺在 `docs/` 根目录。
2. 重大技术决策单独写 ADR，并同步更新 `docs/.ccb/index/decisions.yaml`。
3. 进行中的任务说明优先写到 `docs/.ccb/specs/active/`，稳定后再沉淀到 Layer 1。
4. 失效或废弃内容不要直接删除，优先移动到 `docs/99_归档/`。
